# RP-1000 — Storage Service (Risto Device OS)

## Reporte de Implementación

**Fecha:** 2026-07-17
**Versión Firmware:** 0.0.1 → 0.0.2

---

### Objetivo
Implementar un servicio de almacenamiento persistente (NVS) encapsulado en `StorageManager`, singleton, con interfaz simple. Prohibir el acceso directo a `Preferences` desde cualquier otro módulo del firmware.

---

### Arquitectura

```
┌─────────────────────────────────────┐
│            Application              │
│  setup() → StorageManager::begin()  │
├─────────────────────────────────────┤
│           StorageManager            │
│  (Singleton)                        │
│  ┌───────────────────────────────┐  │
│  │      Preferences (NVS)       │  │
│  │  Namespace: "risto"          │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

- **Patrón:** Singleton (Meyer's Singleton)
- **Backend:** NVS (Non-Volatile Storage) vía `Preferences` de Arduino
- **Namespace:** `risto`
- **Thread safety:** No aplica (single-core loop)

```cpp

// Ciclo de vida

// Consulta y eliminación
bool exists(const char* key);

// Setters / Getters
```

---

### Pruebas

Durante `begin()` se ejecuta una prueba temporal automatizada:

```
Paso 1: setString("device.test", "OK")   → Escribir en NVS
Paso 2: getString("device.test")         → Leer desde NVS
Paso 3: remove("device.test")            → Limpiar dato de prueba
```
